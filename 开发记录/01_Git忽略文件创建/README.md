# Git 忽略文件创建

## 功能描述

为肇庆旅游小程序项目创建完整的 Git 忽略文件配置，确保版本控制中只包含必要的源代码和文档，排除编译输出、依赖库、本地配置和敏感信息。

### 解决的问题
- 避免将编译生成的文件（如 Java 的 target/ 目录）提交到 Git
- 防止敏感信息（如数据库密码、API 密钥）泄露
- 减少仓库体积，提高克隆和拉取速度
- 保持仓库整洁，只包含源代码和必要文档

### 业务价值
- 提升代码安全性
- 规范团队协作流程
- 降低新成员上手成本

---

## 实现逻辑

### 修改的核心文件列表
1. `/肇庆旅游项目/.gitignore` - 项目根目录的 Git 忽略配置
2. `/肇庆旅游_后端/.gitignore` - Java Spring Boot 后端项目专用配置
3. `/肇庆旅游_前端/.gitignore` - 微信小程序前端项目专用配置

### 使用的算法或设计模式
- **分层配置策略**：根目录配置通用规则，子目录配置特定技术栈规则
- **最小化原则**：只忽略必要的文件，避免过度忽略导致文件丢失

### 关键逻辑说明

#### 1. 根目录 .gitignore
- 包含系统文件、IDE 配置、日志文件等通用忽略规则
- 使用通配符匹配后端和前端的特定目录

#### 2. 后端 .gitignore
- Maven/Gradle 构建输出（target/、build/）
- 本地配置文件（application-local.yml 等）
- IDE 配置文件（.idea/、*.iml）

#### 3. 前端 .gitignore
- Node.js 依赖（node_modules/）
- 微信小程序自动生成文件（miniprogram_npm/）
- 敏感配置文件（project.private.config.json）
